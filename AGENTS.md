# 项目规则 (Codex / GPT)

本文件定义了此仓库的硬约束和操作规范。并保持此文件专注于工程约束。

## 不可协商的响应格式
- 始终使用简体中文进行响应。
- 每条助手消息必须以以下内容开头：
  1) `【（必须填写本次实际使用的模型名称）】`
  2) `亲爱的 Wang`

## 范围 / 允许的修改
- 允许：修改后端（Flask）代码，添加新的小程序页面，添加新的后端端点，重构代码以提高可维护性。
- 保持兼容性：小程序和网页必须共享相同的数据和语义，包括收藏/错误/用户答案/用户进度/考试。

## 编码风格和结构
- 保持变更最小化和针对性；避免无关的重构。
- 倾向于模块化设计：避免巨大的文件；在适当的情况下按领域/服务/工具进行拆分。
